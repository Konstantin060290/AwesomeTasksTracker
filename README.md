# Microservices Architecture (Educational Project)

Учебный проект по микросервисной архитектуре.
Проект демонстрирует принципы построения распределённой системы с асинхронным взаимодействием сервисов через брокер сообщений.

## Описание

Система состоит из нескольких независимых сервисов, каждый из которых отвечает за свою бизнес-область.
Взаимодействие между сервисами реализовано через брокер сообщений RedPanda, что обеспечивает слабую связанность и масштабируемость.

## Сервисы
### Billing Service

Сервис начисления и списания средств:

Обработка операций пополнения

Списание средств

Публикация событий об изменении баланса

### Analytics Service

Сервис аналитики:

Подписка на события финансовых операций

Агрегация данных

Формирование статистики

### Authentication Service

Сервис аутентификации:

Регистрация пользователей

Авторизация

### Взаимодействие сервисов

Асинхронная коммуникация через брокер сообщений

Публикация и обработка событий

Отсутствие прямых HTTP-зависимостей между сервисами

## Технологии

ASP.NET MVC

RedPanda (Kafka-совместимый брокер сообщений)

Принципы микросервисной архитектуры

## Цель проекта

Практика проектирования микросервисной архитектуры

Освоение событийного взаимодействия

Разделение ответственности между сервисами

Работа с брокерами сообщений

## Архитектурные принципы

Single Responsibility per Service

Event-Driven Architecture

Проект носит учебный характер и предназначен для демонстрации архитектурных подходов.
